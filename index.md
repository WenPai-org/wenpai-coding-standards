# 前言

## 为什么制定了本规范

每个开发者都有自己的习惯和风格，在独自负责项目时任何习惯和风格只要有利于开发和维护就是好的。

但当多个开发者协同开发项目时，各种风格杂糅在一起就会大大增加项目的维护和协调成本。

本规范继承WordPress官方编码规范的同时进行了一些补充，期望可以让多个开发者协作出来的项目如同出自同一个人之手，以求让你方便的理解别人的代码的同时别人也能轻松理解你的。
